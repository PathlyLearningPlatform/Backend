# Strategy

For working with git, github flow strategy will be used. Github Flow involves working with master branch which will contain working, ready to deploy code. New features will be developed on feature branches which will be branched off master. After work on a specific feature branch is done code will be reviewed, tested and merged into master branch. After that feature branch will be deleted.
